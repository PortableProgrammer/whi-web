Switcher is based on Switcher 2.0 by Bao Nguyen, last updated in late 2007. We've been using it around here since then, and we love it. No other application on the market comes as close to a pure Exposé experience with a minimum of fluff.

History

The original is a wonderful product, and works very well, but it has a few issues. After two years' worth of repeated attempts at submitting bug reports, feature requests, and just generally trying to get Bao to return our messages, we decided to fix the app ourselves.

The end result is forked from Bao's 2.0 release and represents, in our opinion, an almost perfect implementation of Exposé on Windows.

Moral Issues (or, How we learned to stop worrying and consider it abandonware)

We debated long and hard about releasing this to the wild, since it is not in any way endorsed or authorized by Bao, but it's been years since the last update and at least years since we started updating it ourselves, so we feel like it's essentially abandonware. Our major argument against it internally has been that, as of late 2013, insentient.net is still up and running, even though email goes unanswered and the blog link is dead, so someone must be paying that bill, and whomever it is probably is in contact with Bao, meaning there's still the chance that it's not abandoned. Regardless, Bao, if you're around, drop us a line and we'll be happy to discuss the project. We'd love to merge our edits with yours and produce what we feel is a killer app.

Problem?

If you get an error stating "Warning: Aero is disabled", you need to enable Aero for Switcher to work. Windows XP and earlier doesn't support Aero; Windows Vista or later is required. Furthermore, Aero is not available on Windows Vista Home Basic or Windows 7 Starter.
If you have a supported Windows Vista or 7 version, choose "Windows Aero" from the list of available color schemes in the control panel:

  1. Open the Control Panel
  2. Underneath "Appearance and Personalization", select "Change the color scheme"
  3. Choose "Windows Aero" and click "OK"
For Windows 8 and later, Aero is automatically enabled if your hardware supports it.

Known Issues

Administrators: Switcher will not trigger if the currently-active application is running as an administrator and Switcher isn't.

Switcher relies on global mouse and keyboard hooks to detect its triggers. By design, these hooks run in the current user context. If you have UAC enabled, this means that your current user context is a normal user, which cannot access the administrative session. This results in Switcher not receiving trigger events if the currently-active application is run as an administrator. This can be mitigated by running Switcher as an administrator all the time, but that requires responding to the UAC prompt upon every launch, which, to us, is infinitely annoying.

We could also resolve the issue by signing our code with a code-signing certificate, but we can't currently justify the cost associated with the code-signing certificate for a freeware product.

An example of this situation in action:
  • 1. Switcher starts without a UAC prompt
  • 2. Open Notepad (normal app) and Computer Mangement (elevated, requiring UAC prompt)
  • 3. Switcher works normally while Notepad or any other window but Computer Management is active
  • 4. While Computer Management is active, Switcher does not receive trigger notifications
If Switcher is restarted as an elevated user ("run as administrator"), it behaves as expected, since it has access to the administrative session, which also has access to the non-admin session.

Offline / Redirected AppData

Saving the settings to an offline or redirected AppData location will intermittently fail due to timing issue in Windows' offline detection. The fix in v.3.2.1 retries saving several times and will notify the user if saving ultimately fails.

Installer Upgrade

If Switcher is running while attempting an upgrade, the installer might fail to shut it down properly, resulting in a corrupted install. Please manually close Switcher before upgrading.
